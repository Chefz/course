Тестирование

1. Spring Boot, для чего применяется, отличия от Spring Framework

Упрощение и ускорение разработки приложений, автоматическая настройка, встроенные инструменты для развертывания. Работа с базами данных, разработка графического интерфейса, обработка сетевых запросов.

2. Опишите предназначение Spring Initializr

Инструмент для инициализации проектов на основе Spring, позволяющий выбирать зависимости и настраивать параметры. Фреймворк для разработки веб-приложений, конструктор для создания бинов, система контроля версий.

3. Опишите основные компоненты Spring Boot

Spring Boot Starter, Auto-Configuration, Spring Boot Actuator. Spring MVC, Hibernate, JPA.

4. Spring Security, Spring Data, Spring MVC

Spring Security - обеспечение безопасности, Spring Data - взаимодействие с базами данных, Spring MVC - создание веб-приложений. Spring Security - взаимодействие с базами данных, Spring Data - создание веб-приложений, Spring MVC - обеспечение безопасности.

5. Опишите аннотации @Service, @Component, @Repository

@Service - для сервисных компонентов, @Component - общая аннотация для всех компонентов, @Repository - для компонентов, отвечающих за взаимодействие с базой данных. @Service - для создания веб-страниц, @Component - только для тестирования, @Repository - для сервисных компонентов.

6. Отличие аннотации @Service от @Component

@Service - специализированная аннотация для сервисных компонентов, расширяет функциональность @Component. @Service - используется только в веб-приложениях, @Component - только для бинов.

7. Опишите подход Dependency Injection

Внедрение зависимостей, при котором объект получает все необходимые зависимости извне. Взаимодействие с базами данных, создание сетевых запросов, обработка исключений.

8. Как применяется Dependency Injection в Spring Boot

Автоматическое внедрение зависимостей, основанное на аннотациях, конструкторах и сеттерах. Ручное внедрение зависимостей в каждом компоненте, использование только сеттеров для внедрения.

Практическое задание

Используя полученные знания по Spring Boot, реализуйте приложение для учета книг в библиотеке с обеспечением HTTP-взаимодействия.

  • Методы для получения, добавления и удаления книг.
  • Метод для формирования отчета о количестве взятых и оставшихся книг.
  • Метод для поиска просроченных книг.

Реализуйте соответствующие контроллеры, сервисы, и модели. Обеспечьте правильную работу с базой данных.
